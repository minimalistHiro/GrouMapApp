## サービス概要

**GrouMap（ぐるまっぷ）** は、中小飲食店向けのスタンプ＋マップ＋発見型プラットフォームアプリです。

- **キャッチフレーズ**: 小さなお店が見つかる地図
- **対象**: BtoB（飲食店オーナー向け）＋ BtoC（一般ユーザー向け）
- **技術スタック**: Flutter (Dart) + Firebase (Auth/Firestore/Functions/Storage/FCM) + Google Maps
- **対応プラットフォーム**: iOS / Android

> コアコンセプト・ビジネス方針・競合比較・収益モデル・KPI定義については [BUSINESS_MODEL.md](BUSINESS_MODEL.md) を参照

## 1. ユーザー向け機能（BtoC）

### 1.1 認証・アカウント管理

| 機能 | 説明 |
|------|------|
| **メール/パスワード認証** | メールアドレスとパスワードによるログイン・新規登録 |
| **Googleサインイン** | Googleアカウントでのソーシャルログイン |
| **Appleサインイン** | Apple IDでのソーシャルログイン |
| **メールOTP認証** | Cloud Functions経由の6桁コードによるメール認証（登録時必須、送信件名プレフィックス: `【ぐるまっぷ】`） |
| **パスワードリセット** | メールアドレスを入力してパスワード再設定メールを送信 |
| **プロフィール設定（初回登録）** | 新規登録時は生年月日・性別のみ入力（メール/Apple登録時はユーザー名も）。表示名は「ユーザー」、ユーザーIDは「user_」+UID先頭8文字がデフォルトで自動設定。都道府県・市区町村・職業・自己紹介・興味カテゴリ・プロフィール画像は後からプロフィール編集で入力可能。メール認証完了後は `users/{uid}` の入力状態を確認し、未入力項目がある場合は `UserInfoView` へ自動遷移 |
| **プロフィール完成度カード** | アカウント設定画面に2段階で表示。Stage 1: 基本プロフィール6項目（表示名・生年月日・性別・都道府県・市区町村・職業）未完成時は「プロフィールを完成させよう」カード（プロフィール画像は任意でゲージ対象外）。完成させると「自己紹介マスター」バッジ（profile_completed / rare）が獲得できる旨をカード内に表示。Stage 2: 基本完成後に興味カテゴリ未設定時は「興味カテゴリを設定しよう」カード。全完了でカード非表示。Stage 1 カードからプロフィール編集画面に遷移した場合は、保存ボタンが「次へ」に変わり、保存完了後に自動で興味カテゴリ設定画面へ遷移するシームレスなフローになっている |
| **プロフィール編集** | 登録済み基本情報の変更、ユーザーID（ユニーク制約・英数字+アンダースコア・3〜20文字）、自己紹介（最大100文字・任意）、職業（8種選択）、アイコン画像のトリミング・アップロード |
| **興味カテゴリ設定** | プロフィール編集から分離した独立画面。46カテゴリからチップ形式で複数選択。基本プロフィール完成後にアカウント設定リストに表示 |
| **パスワード変更** | 現在のパスワードを確認した上で新パスワードに変更 |
| **メールアドレス変更** | パスワード再認証後、新メールアドレスに6桁OTP認証コードを送信し、検証後にFirebase Auth/Firestoreのメールを更新（メール/パスワード認証のアカウントのみ） |
| **アカウント削除（退会）** | `isActive=false` + `deletedAt` をセットして退会済みとしてマーク。Firebase Auth アカウントのみ削除し、Firestoreデータ（サブコレクション・プロフィール画像含む）は保持。退会フローは `HelpView`（ヘルプ・サポート）の「退会するには？」FAQから `AccountDeletionReasonView`（退会理由10文字以上）へ遷移し、理由送信後に `AccountDeletionProcessingView` で実行 |
| **利用規約・プライバシーポリシー同意** | 登録前の段階同意画面。利用規約/プライバシーポリシーをそれぞれ白枠カードで表示し、各「確認する（必須）」から文書画面へ遷移。文書画面は最下部までスクロールすると下部固定「同意する」が有効化され、押下して戻ると該当項目のみ同意済みに更新。同意済みカードの確認ボタンはグレーアウトし、2項目とも同意済みになった時のみ「同意して次へ」で新規登録へ進める |
| **ウェルカムトップ画面** | 未ログイン起動時に `WelcomeView` を表示し、`splash_logo` とブランドコピーを表示。「新規アカウント登録」「ログイン」「ログインせずに開始（青字・太字テキストボタン）」の3導線を提供 |
| **ゲストモード** | `WelcomeView` の「ログインせずに開始」から、ログインなしでもホーム・マップ・投稿の閲覧が可能（ミッションフローティングボタンは非表示） |
| **オンボーディングチュートリアル** | 新規アカウント登録後、`UserInfoView` 完了直後にチュートリアル（4ページスライド: 地図発見・スタンプ・コイン・バッジ）を全画面表示（保険としてホーム初回遷移時にも `showTutorial=true` なら表示可能）。× スキップ / → 次へ / 「はじめる」で閉じると Firestore `users/{uid}.showTutorial = false` を更新し、次回以降は非表示 |
| **インタラクティブウォークスルー** | チュートリアル完了後に自動開始される6ステップの画面誘導。心理学的理論（情報ギャップ・ツァイガルニク効果・可変報酬・FOMO・自己決定理論）を活用したメッセージ設計で、新規ユーザーが最初の数分でアプリの楽しさ・価値を体感できるよう設計。フロー: ①コンセプト紹介（フルスクリーン・3アイコン説明）→②マップタブをタップ→③店舗マーカーをタップ→④NFCタッチ説明（フルスクリーン・「次へ」ボタン）→⑤図鑑タブをタップ→⑥アカウントタブをタップ（完了）。各ステップにサブテキスト（補足説明）あり。完了/スキップ時に Firestore `users/{uid}.walkthroughCompleted = true` を更新し、次回以降は非表示。ウォークスルー開始セッション中は「今日のレコメンド」ポップアップ・バッジ獲得ポップアップを非表示にし、ウォークスルーと重複表示しない。 |

### 1.2 ホーム画面

| 機能 | 説明 |
|------|------|
| **おすすめ店舗カード** | 候補は `isActive=true` かつ `isApproved=true` の店舗から選定し、`stores.isOwner=true` は除外。表示優先順位は「未訪問（スタンプ0）」→「開拓中（スタンプ1〜9）」→「達成済み（スタンプ10以上）」のフォールバック方式。最大5件を「今日のレコメンド」で1ページ1店舗の全幅カード表示（3秒自動送り・操作中停止）し、状態表示は「未訪問 / スタンプ X/10 / 満了」、距離未取得時は `--` 表示 |
| **ニュースセクション** | 掲載期間内のニュースを画像のみ1:1横スクロールで最大7件表示 |
| **特別クーポンセクション（廃止）** | ※コインシステム廃止に伴い非表示。コイン交換クーポンの新規発行を停止し、UIから削除 |
| **クーポンセクション（廃止）** | ※コインシステム廃止に伴いショートカット導線を非表示 |
| **投稿セクション** | Instagram投稿と通常投稿を日付順で混合プレビュー表示。投稿カードはタイトル非表示・テキスト2行表示。Instagram投稿は「Instagram」バッジ（ピンク）、通常投稿は店舗ジャンルバッジ（オレンジ）を表示 |
| **デイリーレコメンド** | その日初めてのアプリ起動時におすすめ店舗を最大3件自動表示。スコアリングアルゴリズム（カテゴリ一致+2/エリア一致+1/未訪問+2/訪問済み−1/1km以内+3/3km以内+2/5km以内+1）で上位を選定。候補はstoresコレクションから最大50件取得（isActive=true, isApproved=true）。`stores.isOwner=true` の店舗は除外。インプレッション・クリックをFirestoreに記録 |
| **統計カプセルバー** | バッジ・スタンプの現在数をおすすめ店舗セクション上にカプセル形式で横並び表示（コンパクトな丸角カプセル、左アイコン + 「数値+ラベル」表記。アイコンは背景透過PNGを使用。Firestoreリアルタイム取得、バッジ数はローカル定義に存在するバッジのみカウント、未ログイン時は `-` 表示。コイン表示はコインシステム廃止に伴い削除） |

| **スロットキャンペーンボタン（廃止）** | ※スロット機能廃止に伴い削除。コードはコメントアウトで保持 |
| **コイン交換ボタン（廃止）** | ※コインシステム廃止に伴い削除。UIから非表示 |
| **ミッションフローティングボタン（廃止）** | ※コインシステム廃止に伴い削除。UIから非表示 |
| **各機能への導線** | 店舗一覧・クーポン・スタンプカード・バッジ・ランキング・紹介等へのメニュー |

### 1.3 マップ機能

| 機能 | 説明 |
|------|------|
| **ゲームUI（ダークマップ）** | Google Mapsスタイルをダークブルー系ゲームスタイルに変更（ベース色 `#0d1b2a`・水系 `#0a2030`・道路 `#1a3a52`・テキスト `#8ec3b9`・POI非表示）。ポケモンGO/Ingressライクな没入感を演出 |
| **プレイヤーHUD** | 画面下部に常時固定表示（店舗情報パネル表示時は自動非表示）。半透明ダーク背景＋シアンボーダーパネルに「発見（店舗数）」「スタンプ数」「ポイント」の3指標を横並び表示。データは `users/{uid}.discoveredStoreCount`/`stampCount`/`totalPoints` から取得 |
| **レーダーアニメーション** | 現在地を中心に3つの同心円（位相ずらし）が20m→200mに拡大しながらフェードアウトするアニメーションをリピート表示。Google Maps `Circle` APIで描画（`radar_0`〜`radar_2`）。`TickerProviderStateMixin` + 2秒ループ `AnimationController` で実装 |
| **マーカー選択リングアニメーション** | 店舗マーカータップ時に `getScreenCoordinate()` でピンのスクリーン座標を取得し、オレンジ色の円（120px）が拡大しながらフェードアウトする選択リングアニメーション（500ms）をFlutterオーバーレイ上に表示 |
| **ゲーム風店舗情報カード** | 半透明ダーク背景（`#101E2E` 90%不透明）＋シアン発光ボーダー（`#00E5FF`）のゲームパネル。画像上にグラジエントオーバーレイ・左上ジャンルバッジ（オレンジ）・右上営業状況バッジ（グリーンドット）。来店ステータスバッジ（未発見/初発見/探索中/常連/レジェンド）を `totalVisits` から算出して表示。詳細ボタンはオレンジグラジェント（`#FF6B35`→`#FF8C42`） |
| **Google Maps表示** | 現在地周辺の登録店舗をマーカーで地図上に表示。現在地はGoogleマップデフォルトの青ドットで表示 |
| **表示対象店舗条件** | `isActive=true` かつ `isApproved=true` の店舗のみ表示し、`stores.isOwner=true` の店舗は除外 |
| **店舗マーカー** | 白丸ピン＋店舗アイコン画像（画像なし時はカテゴリアイコン）。通常モードではピンアクセント色（ボーダー＋尾）が営業中=緑 `#43A047`・営業時間外=グレー `#BDBDBD` で色分けされる。左上の凡例カードで色の意味を案内 |
| **開拓状況モード** | フィルターチップ「開拓状況」選択時に、ピンのボーダー色が開拓済み（スタンプ1以上）=青 `#2196F3`、未開拓（スタンプ0）=グレー `#BDBDBD` に自動切り替え。チップ解除でリセット（旧「開拓・未開拓」トグルを統合） |
| **店舗検索** | 折りたたみ式検索アイコンボタン（左上）。タップで検索フィールドが展開し、店舗名をテキスト検索で絞り込み。←ボタンまたはクリアで折りたたみ状態に戻る |
| **店舗情報パネル** | マーカータップで店舗の基本情報をパネル表示 |
| **マップ閲覧報酬の上限** | `users/{uid}/daily_missions/{yyyy-MM-dd}.map_open` を日次フラグとして使用し、マップ関連報酬（新規登録ミッション更新・`mapOpened` バッジカウント）は1日1回のみ加算 |
| **エリアオーバーレイ（フェーズ2追加）** | `areas` コレクション（`isActive=true`）から取得したエリア情報を Google Maps の `Circle` ウィジェットで地図上に半透明円として描画。各エリアには中心座標・半径（メートル）・HEXカラーコードを設定し、fillColor（15%透過）・strokeColor（60%透過）で表示。コミュニティマップモードの「開拓率」サブビューで表示 |
| **個人モード（フェーズ2追加）** | マップ上部のモードトグル「個人」選択時に、ピン中心に常に店舗アイコン画像（画像なし時はカテゴリアイコン）を表示し、ピンアクセント色（ボーダー＋尾）を `users/{uid}/stores/{storeId}.totalVisits`（累計来店回数）に応じて5段階で変化させる。未来店=グレー / 初発見（1回）=ライトブルー / 探索中（2〜4回）=グリーン / 常連（5〜9回）=オレンジ / レジェンド（10回〜）=ゴールドグラデーション（金ピカ） |
| **コミュニティモード（フェーズ2追加）** | マップ上部のモードトグル「コミュニティ」選択時に、サブモードトグルで「開拓率」（エリアCircleオーバーレイ）と「賑わい度」（`stores.totalVisitCount` に基づく店舗別Circleオーバーレイ：1〜10件=ライトブルー/半径40m、11〜30件=グリーン/60m、31〜100件=オレンジ/80m、101件〜=ゴールド/100m）を切り替えて表示する |
| **マップモードトグル** | 検索バー直下に常時表示する3択ピルUI（通常/個人/コミュニティ）。選択中モードはAppUi.primaryで背景ハイライト。アプリの醍醐味として前面に配置 |
| **近接自動フォーカス** | Geolocator位置ストリームで5m移動ごとに現在地を更新し、店舗の半径50m以内に入ると自動でそのピンを拡大・店舗情報パネルを表示（ピンタップと同じ挙動）。同時に端末を軽く振動（`HapticFeedback.mediumImpact()`）させる。100m以上離れるとリセットされ、再来接近時に再発火する |
| **現在地ボタン** | 現在位置に地図を移動 |
| **北向き整列** | ワンタップで地図を北向きに揃える |
| **フィルター設定** | 以下の条件で店舗をフィルタリング（設定はFirestoreに保存） |

### 1.5 NFCチェックイン（セルフチェックイン）

| 機能 | 説明 |
|------|------|
| **NFCタッチチェックイン** | 店舗レジ横のNFCタグにスマホをかざすだけでチェックイン完了。NFCタグにはURL型NDEFレコード（`https://groumapapp.web.app/checkin?storeId={storeId}&secret={tagSecret}`）を書き込み、Universal Links（iOS）/ App Links（Android）でアプリが起動。未インストール時は `/checkin` のWebページを表示し、手動でアプリ起動（`groumap://checkin?...`）またはストア遷移を選択可能 |

| **エラーハンドリング** | `FirebaseFunctionsException`のエラーコード別メッセージをダイアログで表示（`already-exists`=本日発見済み / `not-found`=無効NFCタグ / `permission-denied`=利用不可 / `unauthenticated`=未ログイン） |

| **Firebase Auth** | ユーザー認証（Google/Apple/メール/匿名） |
| **Cloud Firestore** | データベース（店舗/ユーザー/トランザクション/クーポン等） |
| **Firebase Storage** | 画像のアップロード・保存（圧縮処理付き） |
| **Cloud Functions** | QRトークン検証、メールOTP送信/検証、レート計算 |
| **Firebase Cloud Messaging** | プッシュ通知の配信 |

### 5.2 主要なCloud Functions

| 関数名 | 機能 |
|--------|------|
| `verifyQrToken` | QRトークンのサーバーサイドJWT検証 |
| `requestEmailOtp` | メール認証用OTPの送信（新規登録時、件名: `【ぐるまっぷ】メール認証コードのお知らせ`） |
| `verifyEmailOtp` | メール認証用OTPの検証（新規登録時） |
| `requestEmailChangeOtp` | メールアドレス変更用OTPの送信（新メールアドレス宛、件名: `【ぐるまっぷ】メールアドレス変更 認証コードのお知らせ`） |
| `verifyEmailChangeOtp` | メールアドレス変更用OTPの検証（Firebase Auth/Firestoreメール更新） |
| `exchangeInstagramAuthCode` | Instagram認可コードの交換（長期アクセストークン取得）と初回投稿同期 |
| `updateInstagramSyncSettings` | 店舗ごとのInstagram定期同期設定（有効/無効、毎日同期時刻）を更新 |
| `syncInstagramPosts` | 指定店舗のInstagram投稿を手動同期（動画除外） |
| `syncInstagramPostsScheduled` | 30分ごとに実行され、同期時刻到達店舗のみInstagram投稿を自動同期 |
| `setStoreOwnerFlagOnCreate` | 店舗作成時にisOwnerユーザーが作成者なら `isOwner: true` を自動設定 |
| `syncStampsWithVisits` | 全店舗のスタンプ数と来店数の不整合チェック・一括同期。不整合ユーザーのdisplayName/profileImageUrl/storeNameをレスポンスに含める（最大50件）。`mode: 'check'` で確認のみ、`mode: 'sync'` で実際に同期実行 |
| `syncStoreOwnerFlags` | 既存店舗のisOwnerフラグ一括同期（ワンタイム実行用Callable） |
| `migrateStampCard` | 物理スタンプカードをデジタルに移行するCallable Function。Firestoreトランザクション内でスタッフ権限確認・二重移行チェック・スタンプ加算・`stamp_migrations` 記録を実行。来店ボーナスは付与しない |
| `notifyFollowersOnNewInstagramPost` | Instagram投稿同期時にフォロワーへプッシュ通知を送信。`public_instagram_posts` の新規ドキュメント作成（`onDocumentCreated`）で発火し、`stores/{storeId}/followers` のフォロワーに通知（動画除外・`notificationSettings.post` 設定を尊重）。再同期時は `merge: true` のため既存ドキュメント更新となりトリガーされず重複通知なし |
| `nfcCheckin` | NFCタグチェックイン処理。タグシークレット検証→店舗確認→1日1回チェックイン制限（`lastStampDate`）→`isFirstVisit` 判定（初回のみ `stores/{storeId}.discoveredCount` を +1）→来店記録（`store_users` / `transactions`）→`store_stats/daily.visitorCount` +1→バッジ判定→自動フォロー（`source:'nfc_checkin'`）→友達紹介コイン処理（維持）→選択クーポン使用処理（6桁確認コード生成）。スタンプ付与・来店ボーナスコイン付与・スタンプ達成クーポン自動付与は廃止。`NfcCheckinResult`（isFirstVisit/storeName/awardedCoupons/usedCoupons/usageVerificationCode）を返す |
| `registerNfcTag` | NFCタグの登録。`nfc_tags/{tagId}`にstoreId・tagSecret・createdAtを保存。管理者オーナー専用 |
| `notifyCouponExpiryScheduled` | コイン交換クーポンの有効期限通知（毎日10:00 JST）。有効期限7日前・3日前のクーポンを検出し、対象ユーザーにお知らせ通知を自動作成。`expiryNotified7d`/`expiryNotified3d` フラグで重複防止 |

### 5.3 データモデル概要

| モデル | 主な用途 |
|--------|---------|
| **StoreModel** | 店舗情報（名前/住所/位置/カテゴリ/営業時間/画像/プラン等） |
| **UserPointBalance（旧）** | 旧ポイント残高モデル（ポイント制度廃止に伴い新規利用停止） |
| **PointTransactionModel（旧）** | 旧ポイント取引モデル（ポイント制度廃止に伴い新規利用停止） |
| **Coupon / UserCoupon** | クーポン定義とユーザーのクーポン取得/使用状況 |
| **BadgeModel / UserBadgeModel** | バッジ定義とユーザーのバッジ取得状況 |
| **NotificationModel** | 通知データ（種別/既読/配信状態/タグ） |
| **RankingModel** | ランキング情報（開拓店舗数/コイン/バッジ/スタンプ/順位変動） |
| **MapFilterModel** | マップフィルター設定 |
| **NewsModel** | ニュース記事（掲載期間制御付き） |
| **PointRequest（旧）** | 旧ポイント付与リクエスト（ポイント制度廃止に伴い新規利用停止） |
| **QRToken / QRVerificationModel** | QRトークンの生成・検証 |
| **OwnerSettings** | プラットフォーム設定（キャンペーン/メンテナンス/バージョン） |
| **FeedbackModel** | フィードバック（ステータス管理付き） |
| **SocialModel** | ソーシャル機能（プロフィール/フォロー/投稿/コメント/いいね） |
