# GrouMap（ぐるまっぷ）サービス機能一覧

> **最終更新**: 2026-03-01（店舗設定詳細の「特別クーポン利用枚数」リネーム＋合計割引額追加、分析画面に全店舗特別クーポン一覧への遷移ボタン追加）

> 本ドキュメントは、ユーザー用アプリ (`groumapapp`) と店舗用アプリ (`groumapapp_store`) の全Dartファイルおよび関連ドキュメントを基に、サービスの全機能をまとめたものです。

---

## サービス概要

**GrouMap（ぐるまっぷ）** は、中小飲食店向けのスタンプ＋マップ＋発見型プラットフォームアプリです。

- **キャッチフレーズ**: 小さなお店が見つかる地図
- **対象**: BtoB（飲食店オーナー向け）＋ BtoC（一般ユーザー向け）
- **技術スタック**: Flutter (Dart) + Firebase (Auth/Firestore/Functions/Storage/FCM) + Google Maps
- **対応プラットフォーム**: iOS / Android

> コアコンセプト・ビジネス方針・競合比較・収益モデル・KPI定義については [BUSINESS_MODEL.md](BUSINESS_MODEL.md) を参照

## 1. ユーザー向け機能（BtoC）

### 1.1 認証・アカウント管理

| 機能 | 説明 |
|------|------|
| **メール/パスワード認証** | メールアドレスとパスワードによるログイン・新規登録 |
| **Googleサインイン** | Googleアカウントでのソーシャルログイン |
| **Appleサインイン** | Apple IDでのソーシャルログイン |
| **メールOTP認証** | Cloud Functions経由の6桁コードによるメール認証（登録時必須、送信件名プレフィックス: `【ぐるまっぷ】`） |
| **パスワードリセット** | メールアドレスを入力してパスワード再設定メールを送信 |
| **プロフィール設定（初回登録）** | 新規登録時は生年月日・性別のみ入力（メール/Apple登録時はユーザー名も）。表示名は「ユーザー」、ユーザーIDは「user_」+UID先頭8文字がデフォルトで自動設定。都道府県・市区町村・職業・自己紹介・興味カテゴリ・プロフィール画像は後からプロフィール編集で入力可能。メール認証完了後は `users/{uid}` の入力状態を確認し、未入力項目がある場合は `UserInfoView` へ自動遷移 |
| **プロフィール完成度カード** | アカウント設定画面に2段階で表示。Stage 1: 基本プロフィール6項目（表示名・生年月日・性別・都道府県・市区町村・職業）未完成時は「プロフィールを完成させよう」カード（プロフィール画像は任意でゲージ対象外）。Stage 2: 基本完成後に興味カテゴリ未設定時は「興味カテゴリを設定しよう」カード。全完了でカード非表示。Stage 1 カードからプロフィール編集画面に遷移した場合は、保存ボタンが「次へ」に変わり、保存完了後に自動で興味カテゴリ設定画面へ遷移するシームレスなフローになっている |
| **プロフィール編集** | 登録済み基本情報の変更、ユーザーID（ユニーク制約・英数字+アンダースコア・3〜20文字）、自己紹介（最大100文字・任意）、職業（8種選択）、アイコン画像のトリミング・アップロード |
| **興味カテゴリ設定** | プロフィール編集から分離した独立画面。46カテゴリからチップ形式で複数選択。基本プロフィール完成後にアカウント設定リストに表示 |
| **パスワード変更** | 現在のパスワードを確認した上で新パスワードに変更 |
| **メールアドレス変更** | パスワード再認証後、新メールアドレスに6桁OTP認証コードを送信し、検証後にFirebase Auth/Firestoreのメールを更新（メール/パスワード認証のアカウントのみ） |
| **アカウント削除（退会）** | `isActive=false` + `deletedAt` をセットして退会済みとしてマーク。Firebase Auth アカウントのみ削除し、Firestoreデータ（サブコレクション・プロフィール画像含む）は保持。退会フローは `HelpView`（ヘルプ・サポート）の「退会するには？」FAQから `AccountDeletionReasonView`（退会理由10文字以上）へ遷移し、理由送信後に `AccountDeletionProcessingView` で実行 |
| **利用規約・プライバシーポリシー同意** | 登録前の段階同意画面。利用規約/プライバシーポリシーをそれぞれ白枠カードで表示し、各「確認する（必須）」から文書画面へ遷移。文書画面は最下部までスクロールすると下部固定「同意する」が有効化され、押下して戻ると該当項目のみ同意済みに更新。同意済みカードの確認ボタンはグレーアウトし、2項目とも同意済みになった時のみ「同意して次へ」で新規登録へ進める |
| **ウェルカムトップ画面** | 未ログイン起動時に `WelcomeView` を表示し、`splash_logo` とブランドコピーを表示。「新規アカウント登録」「ログイン」「ログインせずに開始（青字・太字テキストボタン）」の3導線を提供 |
| **ゲストモード** | `WelcomeView` の「ログインせずに開始」から、ログインなしでもホーム・マップ・投稿の閲覧が可能（ミッションフローティングボタンは非表示） |
| **オンボーディングチュートリアル** | 新規アカウント登録後、`UserInfoView` 完了直後に4枚スライドのチュートリアルを全画面表示（保険としてホーム初回遷移時にも `showTutorial=true` なら表示可能）。地図発見・スタンプ・コイン・バッジの主要機能を紹介。× スキップ / → 次へ / 「はじめる」で閉じると Firestore `users/{uid}.showTutorial = false` を更新し、次回以降は非表示 |

### 1.2 ホーム画面

| 機能 | 説明 |
|------|------|
| **おすすめ店舗カード** | 候補は `isActive=true` かつ `isApproved=true` の店舗から選定し、`stores.isOwner=true` は除外。表示優先順位は「未訪問（スタンプ0）」→「開拓中（スタンプ1〜9）」→「達成済み（スタンプ10以上）」のフォールバック方式。最大5件を「今日のレコメンド」で1ページ1店舗の全幅カード表示（3秒自動送り・操作中停止）し、状態表示は「未訪問 / スタンプ X/10 / 満了」、距離未取得時は `--` 表示 |
| **ニュースセクション** | 掲載期間内のニュースを画像のみ1:1横スクロールで最大7件表示 |
| **特別クーポンセクション** | コイン交換で獲得した100円引きクーポンを黄色テーマの横スクロールカードで表示（1枚以上保有時のみ・枚数バッジ付き・タップでクーポン詳細へ遷移） |
| **クーポンセクション** | 利用可能なクーポンの一覧へのショートカット |
| **投稿セクション** | Instagram投稿と通常投稿を日付順で混合プレビュー表示。投稿カードはタイトル非表示・テキスト2行表示。Instagram投稿は「Instagram」バッジ（ピンク）、通常投稿は店舗ジャンルバッジ（オレンジ）を表示 |
| **デイリーレコメンド** | その日初めてのアプリ起動時におすすめ店舗を最大3件自動表示。スコアリングアルゴリズム（カテゴリ一致+2/エリア一致+1/未訪問+2/訪問済み−1/1km以内+3/3km以内+2/5km以内+1）で上位を選定。候補はstoresコレクションから最大50件取得（isActive=true, isApproved=true）。`stores.isOwner=true` の店舗は除外。インプレッション・クリックをFirestoreに記録 |
| **統計カプセルバー** | コイン・バッジ・スタンプの現在数をおすすめ店舗セクション上にカプセル形式で横並び表示（コンパクトな丸角カプセル、左アイコン + 「数値+ラベル」表記。アイコンは背景透過PNGを使用。Firestoreリアルタイム取得、バッジ数はローカル定義に存在するバッジのみカウント、未ログイン時は `-` 表示） |
| **ホームショートカットメニュー** | スタンプ/バッジ/店舗一覧/クーポンを4つの独立した白い円形ボタンで表示。各ボタン内に背景透過したオレンジ系アイコンを配置し、下部にラベルを表示。タップで各機能へ遷移 |
| **スロットキャンペーンボタン（廃止）** | ※スロット機能廃止に伴い削除。コードはコメントアウトで保持 |
| **ミッションフローティングボタン** | ログインユーザーのみ表示。72x72の小型円形ボタン。受取可能なミッション（達成済み＆未受取）がある場合は黄色系グラデーション、ない場合はグレーアウト表示。影は従来より薄め。新規登録ミッション未完了時はデイリー・ログインボーナスの受取判定をスキップし誤点灯を防止。ミッション画面から戻った時に状態を再判定 |
| **各機能への導線** | 店舗一覧・クーポン・スタンプカード・バッジ・ランキング・紹介等へのメニュー |

### 1.3 マップ機能

|------|------|

### 1.4 店舗詳細

| 機能 | 説明 |
|------|------|
| **上部ヘッダー** | 店舗イメージ画像 + 左下重ねアイコン + 店舗基本情報（店名/お気に入り/フォロー/営業状態/カテゴリ/住所/営業時間/店舗説明）をタブ上に共通表示 |
| **トップタブ** | 利用可能クーポン一覧、投稿プレビュー（新着15件・3列グリッド） |
| **店内タブ** | 店内画像のギャラリー表示 |
| **メニュータブ** | カテゴリ別フィルタ（コース/料理/ドリンク/デザート）でメニュー一覧を表示（画像・価格付き） |
| **投稿タブ** | Instagram投稿と通常投稿を混合表示（動画除外・最大51件・日付降順） |
| **お気に入り操作** | 店舗をお気に入りに追加/解除 |
| **店舗フォロー** | 店舗をフォロー/解除。フォロー中の店舗が投稿・クーポンを発行するとプッシュ通知を受信。来店（スタンプ獲得）時に自動フォロー |

| **営業時間表示（週別）** | 今日〜6日後の7日間を日付付きで表示（例: `2/24(火)`）。`scheduleOverrides` 優先で臨時休業・通常営業（不定休時）・臨時営業・時間変更を反映。今日の行はオレンジハイライト、特別スケジュール日にはバッジ（臨時/変更）を表示。不定休（`isRegularHoliday=true`）の場合: type='open' override がある日は「通常営業 HH:mm〜HH:mm」（バッジなし）、それ以外は「定休日」を表示 |
| **スケジュール変更バナー** | 今後14日以内に臨時休業・時間変更・臨時営業（不定休時は通常営業）がある場合、営業時間セクション上部に黄色バナーで一覧表示 |
| **決済方法表示** | 対応している決済方法をカテゴリ別にChip表示（現金/カード/電子マネー/QR決済） |
| **座席数** | テキストで座席情報を表示 |
| **設備・サービス情報** | アクセス情報、駐車場/テイクアウト/喫煙/Wi-Fi/バリアフリー/子連れ/ペットの対応状況をChip表示（利用可能=オレンジ/不可=グレー） |
| **SNS・ウェブサイトリンク** | 設定済みのサービスのみ円形アイコン（直径48px）を左から横並びで表示（Web/X/Instagram/Facebook の順）。タップで外部ブラウザまたは各アプリを起動（`url_launcher`）。`font_awesome_flutter` パッケージのブランドアイコンを使用。未設定サービスは非表示 |

### 1.6 スタンプ

| 機能 | 説明 |
|------|------|
| **スタンプカード一覧** | ユーザーが持つ各店舗のスタンプカードを一覧表示 |
| **スタンプ獲得** | 来店時のQRチェックインでスタンプを獲得 |
| **スタンプカード運用** | 累積型（リセットなし）。スタンプ数は来店のたびに加算し続け、10個達成ごとに「○枚目」へ進む（枚数制限なし）。カードには「N枚目」紫バッジを1枚目から常時表示し、現在のカード番号と進捗（N/10）を一目で確認できる |
| **スタンプ達成特典（自動付与）** | 10の倍数到達ごとに、店舗が任意設定したクーポン（`requiredStampCount > 0`）を自動付与。有効期限なし・累積保有可能。`user_coupons` に `type: 'stamp_reward'` で保存 |
| **スタンプ達成クーポンUI** | ホーム「特別クーポン」セクションにコイン交換クーポンと合わせて表示。クーポン詳細画面でスタンプ達成バッジを表示。使用は店舗フロー（QRスキャン時）のみ |
| **スタンプ履歴** | 全て/獲得履歴のタブで時系列表示 |
| **取引履歴** | クーポン利用履歴とスタンプ受取導線を表示 |
| **スタンプ押印申請** | 手動でスタンプ押印を店舗に申請 |
| **付与結果画面** | スタンプ獲得演出、使用/未使用クーポンのリスト表示 |

### 1.7 クーポン

| 機能 | 説明 |
|------|------|
| **クーポン一覧** | 利用可能/使用済みのタブ切替で表示 |
| **クーポン詳細** | ヘッダー画像、概要、期限、割引情報、必要スタンプ数、店舗情報、「クーポンのご利用方法」3ステップガイド（①店舗スタッフに伝える→②QRコード提示→③店舗がスキャンして適用）、注意事項（不正利用防止のため店舗側スキャン時のみ適用される旨）を表示 |

### 1.8 バッジ・ゲーミフィケーション

| 機能 | 説明 |
|------|------|
| **バッジ一覧** | 全162種のバッジをグリッド表示（うちスロット関連8個は廃止・獲得不可）。Firestoreの `user_badges` から取得済みバッジを判定し、取得済みは画像・名前を通常表示、未取得は「？」アイコン・「？？？」名前で表示。ヘッダー右上フィルターでカテゴリ絞り込み可能 |
| **バッジ獲得演出** | バッジ獲得時のアニメーション演出画面（ホーム画面表示2秒後にポップアップ。レコメンドポップアップとの競合時はレコメンド優先→閉じた後2秒でバッジ表示。セッション内で同一バッジの重複表示防止済み。ローカル定義に存在するバッジのみ表示） |
| **バッジ進捗管理** | 各バッジの達成条件に対する進捗を表示 |

## 5. 技術基盤

### 5.1 バックエンド（Firebase）

| サービス | 用途 |
|---------|------|
| **Firebase Auth** | ユーザー認証（Google/Apple/メール/匿名） |
| **Cloud Firestore** | データベース（店舗/ユーザー/トランザクション/クーポン等） |
| **Firebase Storage** | 画像のアップロード・保存（圧縮処理付き） |
| **Cloud Functions** | QRトークン検証、メールOTP送信/検証、レート計算 |
| **Firebase Cloud Messaging** | プッシュ通知の配信 |

### 5.2 主要なCloud Functions

| 関数名 | 機能 |
|--------|------|
| `verifyQrToken` | QRトークンのサーバーサイドJWT検証 |
| `requestEmailOtp` | メール認証用OTPの送信（新規登録時、件名: `【ぐるまっぷ】メール認証コードのお知らせ`） |
| `verifyEmailOtp` | メール認証用OTPの検証（新規登録時） |
| `requestEmailChangeOtp` | メールアドレス変更用OTPの送信（新メールアドレス宛、件名: `【ぐるまっぷ】メールアドレス変更 認証コードのお知らせ`） |
| `verifyEmailChangeOtp` | メールアドレス変更用OTPの検証（Firebase Auth/Firestoreメール更新） |
| `exchangeInstagramAuthCode` | Instagram認可コードの交換（長期アクセストークン取得）と初回投稿同期 |
| `updateInstagramSyncSettings` | 店舗ごとのInstagram定期同期設定（有効/無効、毎日同期時刻）を更新 |
| `syncInstagramPosts` | 指定店舗のInstagram投稿を手動同期（動画除外） |
| `syncInstagramPostsScheduled` | 30分ごとに実行され、同期時刻到達店舗のみInstagram投稿を自動同期 |
| `setStoreOwnerFlagOnCreate` | 店舗作成時にisOwnerユーザーが作成者なら `isOwner: true` を自動設定 |
| `syncStampsWithVisits` | 全店舗のスタンプ数と来店数の不整合チェック・一括同期。不整合ユーザーのdisplayName/profileImageUrl/storeNameをレスポンスに含める（最大50件）。`mode: 'check'` で確認のみ、`mode: 'sync'` で実際に同期実行 |
| `syncStoreOwnerFlags` | 既存店舗のisOwnerフラグ一括同期（ワンタイム実行用Callable） |
| `migrateStampCard` | 物理スタンプカードをデジタルに移行するCallable Function。Firestoreトランザクション内でスタッフ権限確認・二重移行チェック・スタンプ加算・`stamp_migrations` 記録を実行。来店ボーナスは付与しない |
| `notifyCouponExpiryScheduled` | コイン交換クーポンの有効期限通知（毎日10:00 JST）。有効期限7日前・3日前のクーポンを検出し、対象ユーザーにお知らせ通知を自動作成。`expiryNotified7d`/`expiryNotified3d` フラグで重複防止 |

### 5.3 データモデル概要

| モデル | 主な用途 |
|--------|---------|
| **StoreModel** | 店舗情報（名前/住所/位置/カテゴリ/営業時間/画像/プラン等） |
| **UserPointBalance（旧）** | 旧ポイント残高モデル（ポイント制度廃止に伴い新規利用停止） |
| **PointTransactionModel（旧）** | 旧ポイント取引モデル（ポイント制度廃止に伴い新規利用停止） |
| **Coupon / UserCoupon** | クーポン定義とユーザーのクーポン取得/使用状況 |
| **BadgeModel / UserBadgeModel** | バッジ定義とユーザーのバッジ取得状況 |
| **NotificationModel** | 通知データ（種別/既読/配信状態/タグ） |
| **RankingModel** | ランキング情報（コイン/バッジ/スタンプ/順位変動） |
| **MapFilterModel** | マップフィルター設定 |
| **NewsModel** | ニュース記事（掲載期間制御付き） |
| **PointRequest（旧）** | 旧ポイント付与リクエスト（ポイント制度廃止に伴い新規利用停止） |
| **QRToken / QRVerificationModel** | QRトークンの生成・検証 |
| **OwnerSettings** | プラットフォーム設定（キャンペーン/メンテナンス/バージョン） |
| **FeedbackModel** | フィードバック（ステータス管理付き） |
| **SocialModel** | ソーシャル機能（プロフィール/フォロー/投稿/コメント/いいね） |
