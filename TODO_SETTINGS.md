#### A-17. ミッション・コイン・クーポン交換設定のFirestore管理化
- **優先度**: 中（運営リスク低減）
- **ビジネスモデル記述**: ミッション報酬・コイン交換レート・クーポン金額などのハードコード値をFirestoreで管理し、アプリアップデートなしで運用変更できるようにする
- **対象**: ユーザー用アプリ + Firestore（owner_settings/current）

- **現在ハードコードされている値**:
  - コイン交換レート: 10コイン（`mission_service.dart:468`）
  - クーポン割引額: 100円（`mission_service.dart:490`）
  - クーポン有効期限: 30日（`mission_service.dart:472`）
  - コイン有効期限: 180日（`coin_service.dart:32`）
  - デイリーミッション報酬: app_open=1, recommendation_view=1, feed_view=1（`missions_view.dart:200-204`）
  - ログインボーナス報酬: login_3=2, login_7=5, login_30=10（`missions_view.dart:237-240`）
  - 新規登録ミッション報酬: profile_completed=3, first_map=10, first_favorite=2, first_store_detail=1, first_stamp=3（`missions_view.dart:274-282`）

- **設計方針**: 既存の `owner_settings/current` ドキュメントにフィールドを追加（新規コレクション不要）。Firestoreルール変更不要（`owner_settings/{storeId}` で `read: true, write: isOwner()` 設定済み）。全フィールドにデフォルト値を設定し、Firestoreに値がなくても現在と同じ動作を維持

  ```
  coinExchangeDiscount: 100         // クーポン割引額（円）
  coinExchangeCouponDays: 30        // クーポン有効期限（日数）

  loginBonusRewards: {              // ログインボーナス報酬
    login_3: 2, login_7: 5, login_30: 10
  }
  registrationMissionRewards: {     // 新規登録ミッション報酬
    profile_completed: 3, first_map: 10, first_favorite: 2,
    first_store_detail: 1, first_stamp: 3
  }
  ```
- **実装ステップ**:
  1. `lib/models/mission_config.dart` 新規作成 — MissionConfigクラス（fromOwnerSettingsファクトリ、全デフォルト値定義）
  2. `lib/providers/mission_config_provider.dart` 新規作成 — ownerSettingsProviderからの派生Provider
  3. `lib/services/coin_service.dart` 修正 — coinExpiryDaysを外部注入可能に（calculateExpiryDate/buildCoinEarnUpdateにexpiryDaysオプション引数追加）
  4. `lib/services/mission_service.dart` 修正 — exchangeCoinForCouponにrequiredCoins/discountValue/couponDaysオプション引数追加
  5. `lib/views/missions/missions_view.dart` 修正 — 全ハードコード値をmissionConfigProvider経由に変更、UIテキストを動的生成
  6. Firestoreにデフォルト値を投入、FIRESTORE.md更新
- **変更しないもの**: Firestoreルール、複合インデックス、Cloud Functions、ミッションID（進捗管理キーのため不変）
- **備考**: ミッションIDは `daily_missions` / `missionProgress` のキーとして使用されているため変更不可。新しいミッションの追加はアプリアップデートが必要（UI定義がコード内にあるため）。将来的に店舗用アプリの `owner_settings_view.dart` にミッション設定管理UIを追加可能
