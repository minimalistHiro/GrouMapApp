# 3タブナビゲーション再設計 設計書
## ユーザー用アプリ（groumapapp）UI刷新

作成日: 2026-03-04

---

## 背景・目的

コインシステムの廃止・ゲーム性（図鑑・地図塗り）への転換に伴い、既存の5タブ構成を3タブに刷新する。

- **廃止**: ホームタブ・投稿タブ・QRフローティングボタン
- **新設**: 図鑑タブ（新規Viewとして作成）
- **強化**: アカウントタブ（通知・ランキング・探検統計を統合）
- **起点変更**: マップを最初のタブ（index 0）に変更

---

## 新しいタブ構成（確定）

```
[ マップ ]  [ 図鑑 ]  [ アカウント ]
  index 0   index 1    index 2
```

### タブ変更対照表

| index | 変更前 | 変更後 |
|-------|--------|--------|
| 0 | HomeView | MapView ★ |
| 1 | MapView | ZukanView（新規作成）★ |
| 2 | QR（FAB） | ProfileView（強化）★ |
| 3 | PostsView | 廃止 |
| 4 | ProfileView | 廃止 |

**ゲストモード**: マップ / 図鑑（ログイン促進表示）/ アカウント（ゲスト表示）の3タブ

## 1. MainNavigationView の変更

**ファイル**: `lib/views/main_navigation_view.dart`

### 変更点

```
// Before: 5タブ
[HomeView, MapView, QR(placeholder), PostsView, ProfileView]

// After: 3タブ
[MapView, ZukanView, ProfileView]
```

- `_buildBottomScaffold()` のタブ定義を3つに縮小
- `floatingActionButton` および `_LoweredFabLocation` を削除
- `floatingActionButtonLocation` を削除
- `_onQrFabPressed()` メソッドを削除
- 初期タブインデックスを `0`（MapView）に固定
- Deep Link受信（NFCチェックイン）は引き続き `MainNavigationView` で処理

```dart
// ゲスト:       [MapView, ZukanView（ログイン促進）, ProfileView（ゲスト表示）]
```

## 2. 図鑑View（新規作成）

**ファイル**: `lib/views/encyclopedia/encyclopedia_view.dart`

### 画面構成

```
┌─────────────────────────────────┐
│  図鑑             [フィルター▼] │  ← CommonHeader
├─────────────────────────────────┤
│  発見済み: 7店舗 / 全18店舗     │  ← 開拓サマリー
├─────────────────────────────────┤
│  [カフェ][ラーメン][和食]...    │  ← カテゴリチップフィルター
├─────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐         │
│  │発見│ │？？│ │発見│         │  ← 店舗カードグリッド（2列）
│  └────┘ └────┘ └────┘         │
│  ┌────┐ ┌────┐ ┌────┐         │
│  │発見│ │？？│ │？？│         │
│  └────┘ └────┘ └────┘         │
└─────────────────────────────────┘
```

### 店舗カードの仕様

**発見済みカード（NFCタッチ済み）**
- 店舗写真（あれば）またはカテゴリアイコンを背景に使用
- 店舗名・カテゴリ・レア度バッジを表示
- タップで店舗詳細へ遷移

**未発見カード**
- グレースケール + 透明度50%処理
- 店舗名を「？？？」で表示
- カテゴリ・エリア・距離は表示する（情報ギャップ理論 — 覗き見させることで探索意欲を喚起）
- レア度バッジは表示する

### レア度バッジ

| レア度 | 条件 |
|--------|------|
| ★★★★ レジェンド | NFCタッチ者数 0人（まだ誰も来ていない） |
| ★★★ エピック | タッチ者数 1〜5人 または 新規開業1ヶ月以内 |
| ★★ レア | タッチ者数 6〜20人 |
| ★ コモン | タッチ者数 21人以上 |

`stores/{storeId}.rarityOverride` があれば手動設定を優先する。

### データ取得

```
Firestore:
- stores: isActive=true, isApproved=true, isOwner=false
- users/{uid}/stores: NFCタッチ済みの storeId 一覧（発見判定）
```

既存の `StampCardWidget` のカテゴリアイコン・カラー定義を流用する（新規アセット作成なし）。

### ゲストモード表示

- ログイン促進カード1枚を上部に固定表示
- 店舗カードは全てシルエット表示（タップでログイン促進ダイアログ）

### 主な変更点

1. **探検統計カード（新規）**
   - 発見店舗数・バッジ数・ランキング順位・エリア開拓率（将来）を表示
   - 既存の `UserStatsCard` を拡張または置き換え

2. **通知ベルアイコン（ヘッダー右上）**
   - 未読通知数のバッジ表示
   - タップで `NotificationsView` へ遷移
   - 現在 `HomeView` ヘッダーにある通知ベルをこちらに移動

3. **メニュー構成の変更**
   - 「バッジ一覧」「ランキング」「通知・お知らせ」を上部の「ゲーム」セクションに追加
   - 「友達を紹介する」を「アカウント設定」セクションに移動（現在のホーム紹介セクションを移植）

4. **QRコードフォールバックボタン**
   - グレーの小さなテキストボタン形式（ログアウトの上）
   - ラベル例: 「QRコードを表示する（NFC非対応端末向け）」
   - タップで `QRGeneratorView` へ遷移

## 4. ホーム画面・投稿タブの扱い

### HomeView
- タブから除外するが**ファイルは削除しない**
- 「おすすめ店舗」ロジックはマップ画面のレコメンドとして将来再利用検討
- ニュースセクションはアカウント内の通知・お知らせに統合

### PostsView
- タブから除外するが**ファイルは削除しない**
- 店舗詳細画面の「投稿タブ」からのアクセスは継続

## 5. ウォークスルーの更新

現在のウォークスルー（4ステップ）を新しいタブ構成に合わせて変更する。

| ステップ | 変更前 | 変更後 |
|---------|--------|--------|
| ① | マップタブをタップ | マップ画面でマーカーをタップ（起動時からマップのため省略 or 変更） |
| ② | マーカータップ | 店舗パネルを確認 |
| ③ | パネルを閉じる | パネルを閉じる |
| ④ | ホームタブに戻る | 図鑑タブに遷移して発見済み店舗を確認 |
