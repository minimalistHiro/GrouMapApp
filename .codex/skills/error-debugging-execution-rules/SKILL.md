# Error Debugging Execution Rules

## 概要

エラーが発生した場合は、自分でターミナル実行して確認と修正まで行う。権限不足が原因の場合も、ユーザーに手動実行させず、承認付きで自分が実行する方針を徹底する。

## 実行ルール

以下を必ず順守する。

1. エラーが発生したら、ターミナルで自分がコマンドを実行し、確認可能な範囲は自分で調査して修正まで行う。
2. 権限不足やネットワーク制限などで実行できない場合も、ユーザーに手動実行させない。
3. 実行権限が必要な場合は、自分がコマンドを実行し、ユーザーが承認を付与する形で進める。
4. ログインが必要な場合はユーザーにログインを依頼し、ログイン後は自分が引き継いで実行・調査・修正を続ける。
